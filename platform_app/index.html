    Technology

    Start with the product story, then inspect the stack.

    The public deployment now summarizes the pipeline first instead of dropping visitors into raw documentation. The detailed architecture remains visible below for technical reviewers.

    Signal path

    3 IMU nodes → sensor fusion → pen tip → recognition

    Forearm, hand, and pen orientation are fused into a stable trajectory that drives both the studio view and ML features.

    Core methods

    Madgwick, 15-state ESKF, forward kinematics, and drift constraints

    The pipeline favors wearable sensing and geometric reconstruction instead of camera calibration or external tracking.

    Deployment split

    Browser demo by default, live relay when local services exist

    Public visitors get a usable studio immediately; local operators can still inspect live websocket sessions from the same app.

    System pipeline

    ESP32 + 3 IMUs UDP packets Madgwick + ESKF Forward kinematics Browser studio + ML APIs

    Hardware

    MPU-6050 sensors on forearm and hand, ICM-20948 on pen, ESP32 transport, and a button for pen-down state.

    Fusion

    Madgwick attitude estimation feeds a 15-state ESKF and biomechanical constraints to stabilize motion.

    Recognition

    Stroke features are extracted from localized trajectories and exposed through the existing `/api/ml/*` routes.

    Interfaces

    HTTP serves the public app and feedback APIs while the websocket relay remains optional for live sessions.

    Install guide For actual users and testers
    1

    Install the Android app

    Preferred path: open the `android_app` project in Android Studio and run it on a device. Alternate path: build an APK and sideload it manually.

    2

    Enable Developer options

    On Samsung or Android devices: Settings → About phone → Software information → tap Build number 7 times.

    3

    Turn on USB debugging

    Settings → Developer options → enable USB debugging. This is required if you install or debug the app from Android Studio.

    4

    Allow app installation

    If you use a built APK instead of Android Studio, allow installation from the file manager or browser that opens the APK.

    5

    Join the same Wi-Fi

    The phone and the PC running `action_dispatcher.py` must be on the same local network for `ws://PC_IP:18800` to work reliably.

    6

    Scan QR or enter IP manually

    The Android app already supports QR scanning and direct IP entry. Use the QR on this page or type the shown IP into the app.

    Android Studio path

    Open the `android_app` project, connect a phone, confirm USB debugging authorization, and press Run. This is the best path while iterating on intents and mappings.

    APK path

    Build a release or debug APK, transfer it to the device, then open it from Downloads or a file manager and allow the installation source if prompted.

    Why developer mode matters

    The app launches Android intents and is easier to test, reinstall, and diagnose when Developer options and USB debugging are enabled during setup.

    Troubleshooting

    If pairing fails, check same-network status, firewall rules, whether port 18800 is open on the PC, and whether `action_dispatcher.py` is currently running.
