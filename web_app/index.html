1. 다중 센서 융합 시스템 (Multi-Sensor Fusion)

시스템은 사용자의 전완(Forearm, S1), 손등(Hand, S2), 펜(Pen, S3)에 각각 부착된 3개의 독립적인 IMU 노드로 구성됩니다. 각 노드는 초당 85Hz의 속도로 가속도(Accelerometer), 자이로스코프(Gyroscope), 자기장(Magnetometer - S3 한정) 원시 데이터를 송출합니다.

V3 Packet Structure (92 bytes)

[0xAA] [4B timestamp] [S1: 24B (6×float)] [S2: 24B (6×float)] [S3: 36B (9×float)] [1B button] [1B checksum] [0x55]

각 센서 데이터는 struct.unpack("<6f") 또는 "<9f"로 파싱되며, 펌웨어 단에서 x=-x, y=-y 축 매핑이 적용됩니다.

2. Madgwick MARG 필터 (Attitude Estimation)

쿼터니언 업데이트: qt = qt-1 + dt × (½ × qt-1 ⊗ ω - β × ∇f / ||∇f||) 여기서: q = 쿼터니언 [w, x, y, z] ω = 자이로스코프 각속도 [rad/s] β = 수렴 속도 파라미터 (기본값: 0.1) ∇f = 가속도 + 지자기 오차의 기울기 (gradient)

펜 끝(S3)에 장착된 ICM-20948 센서로부터 자기장(Magnetometer) 데이터를 수신하여, Madgwick MARG (Magnetic, Angular Rate, and Gravity) 필터를 구동합니다. S1/S2는 6축 IMU 모드로 동작합니다.

  • 수렴 대기(Warmup): 최초 200 프레임 동안 필터가 안정화될 때까지 FK 연산을 보류합니다.
  • Yaw 드리프트 억제: 9축 MARG 모드에서 지구 자기장 벡터를 활용하여 절대 방위각을 확보하며, 수평 회전축 드리프트를 제거합니다.

3. 15-State Error-State Kalman Filter (ESKF)

State Vector δx (15차원)

δx = [δp(3), δv(3), δθ(3), b_g(3), b_a(3)] δp : 위치 오차 (Position Error) [m] δv : 속도 오차 (Velocity Error) [m/s] δθ : 자세 오차 (Attitude Error, Rodrigues Parameterization) [rad] b_g : 자이로스코프 바이어스 (Gyroscope Bias) [rad/s] b_a : 가속도계 바이어스 (Accelerometer Bias) [m/s²]

ESKF는 가속도를 이중 적분하여 위치를 추산합니다. 드리프트를 억제하기 위해 총 7개의 보정 레이어가 사용됩니다:

  1. 가속도 바이어스 보정 (Nominal Predict)
  2. ZUPT — Zero Velocity Update (에너지 기반 SHOE 검출기)
  3. ZARU — Zero Angular Rate Update (정지 시 자이로 보정)
  4. Adaptive Process Noise Q (운동 의존적 노이즈 스케일링)
  5. Velocity Decay (가속도 미감지 시 지수적 속도 감쇠)
  6. Velocity Clamping (물리적 한계 3 m/s)
  7. FK Pseudo-Measurement (외부 적용)

4. 순방향 운동학 (Forward Kinematics)

Ptip = Origin + R(S1q) × Lforearm + R(S2q) × Lhand + R(S3q) × Lfinger Skeleton Chain: Origin(0,0,0) →[S1: 25cm]→ Elbow →[S2: 18cm]→ Wrist →[S3: 8cm]→ Pen Tip BONE_DIR = [0, 1, 0] (Y-Forward)

각 센서의 Madgwick 쿼터니언 출력을 회전 행렬 R로 변환한 뒤, 뼈대 방향 벡터와 곱하여 순차적으로 관절 좌표를 도출합니다. 가속도 이중 적분 없이 순수 기하학적 위치를 계산하므로 드리프트가 원천적으로 발생하지 않습니다.

5. Writing Plane Constraint (2.5D 칠판 모드)

ESKF 가속도의 Y축(깊이) 성분을 억제하여 XZ 평면(좌/우 × 위/아래)에 필기를 강제합니다.

  • PCA 모드: 최근 100개 위치로 주성분 분석하여 자동으로 필기 평면 법선 벡터를 탐지.
  • Absolute Lock 모드: 법선을 Y축 [0,1,0]으로 고정하여 깊이 방향 드리프트를 완전 차단.

6. Zero Velocity Update (ZUPT)

SHOE Detector (Stance Hypothesis Optimal Estimation)

Energy = (1/W) × Σ [||a_i - g||² / σ_a² + ||ω_i||² / σ_g²] If Energy < Threshold → ZUPT Active: v=0, Kalman Update

슬라이딩 윈도우(W=20) 내 관성 에너지가 임계값 이하일 경우 정지 상태로 판정하여 속도를 강제로 0으로 교정합니다. 이를 통해 시간에 비례하여 발산하는 위치 드리프트를 주기적으로 리셋합니다.

하드웨어 설계 및 센서 스펙 (Hardware)

AirWriting 하드웨어는 초소형, 저지연, 고성능 네트워크 전송을 목표로 설계된 커스텀 임베디드 디바이스입니다.

1. 메인 컨트롤러 (MCU)

ESP32 (NodeMCU)

  • Architecture: Xtensa Dual-Core 32-bit LX6 Microprocessor
  • Clock Speed: 240 MHz
  • Network: 2.4 GHz 802.11 b/g/n Wi-Fi, UDP Socket 데이터 스트리밍 (85~100 FPS)
  • I2C MUX 대용: ESP32의 Wire(0) 및 Wire1(1) 하드웨어 듀얼 I2C 버스를 활용하여 주소 충돌(0x68) 센서를 물리적으로 격리

2. 센서 노드 (IMU 네트워크)

S1 & S2: MPU-6050 (6-DOF)

  • Mounting: 전완(Forearm) 및 손등(Hand) 부착
  • Sensors: 3축 Accelerometer (±8g), 3축 Gyroscope (±1000 dps)
  • Interface: I2C 분리 버스 (GPIO 21/22, GPIO 32/33)
  • Role: 베이스 팔 관절의 롤/피치 회전 각도 연산

S3: ICM-20948 (9-DOF)

  • Mounting: 펜(Pen Tip) 끝부분 부착
  • Sensors: 3축 Accel (±8g), 3축 Gyro (±2000 dps), 3축 Magnetometer (AK09916, ±4900 µT)
  • Interface: I2C (Address 0x69)
  • Role: 메인 End-Effector. 지자기 센서 융합 + 펜 버튼 (GPIO 15)

본 시스템은 UWB 앵커 인프라 없이 순수 IMU 3개만으로 98.1%의 인식률을 달성하며, 하드웨어 비용 대비 최고 효율을 달성. 카메라 기반 시스템($200+)과 비교 시 1/5 이하의 비용으로 동등 성능 확보.

ESKF vs. FK — 궤적 비교

동일 필기("A") 시 ESKF 가속도 적분 vs. FK 쿼터니언 기반 궤적

ESKF 방식은 가속도 이중적분의 누적 오차로 인해 수직 이동(Z축) 궤적이 시간에 따라 급격히 발산합니다. 반면 FK 방식은 쿼터니언 회전 행렬의 기하학적 체인만을 사용하므로 드리프트가 원천적으로 0입니다.

ESKF 궤적: 가속도 → 속도(∫) → 위치(∫∫) → 누적 오차 O(t²) FK 궤적: 쿼터니언 → 회전행렬 → 관절체인곱 → 오차 = 0 (기하학적) 결론: FK 모드에서 상하(Z축) 움직임이 완벽하게 보존됨 (digital_twin.py와 동일한 움직임 확인)

Phase 4: ML 문자 인식 모델

모델 아키텍처 후보

Option A: LSTM (Long Short-Term Memory) Input: 시퀀스 [T×F] (T=타임스텝, F=특징 차원) Hidden: 128 units × 2 layers (Bidirectional) Output: Softmax(26) → A-Z 확률 분포 Option B: 1D-CNN + GRU Conv1D(64, kernel=3) → Conv1D(128) → GRU(64) → Dense(26) 장점: 로컬 패턴(획) 포착에 강함 Option C: Transformer Encoder Positional Encoding → 4-Head Attention × 3 layers 장점: 긴 시퀀스의 글로벌 의존성 학습

Phase 5: 응용 서비스 구상

  • 실시간 영어 단어 인식: 글자 단위 인식 결과를 사전(Dictionary)과 매칭하여 자동완성 제안
